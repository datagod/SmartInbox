Gmail (IMAP + App Password)

Checking…

Before connecting: In Gmail web mail, go to Settings → See all settings → Forwarding and POP/IMAP and choose Enable IMAP. Then create a Google App Password (requires 2-Step Verification). Use the 16-character app password — not your regular Gmail password. Stored locally in data/smartinbox.db only.

Proton Mail (Bridge + IMAP)

Checking…

Proton Mail requires Proton Mail Bridge running on this machine. In Bridge, sign in, enable IMAP, and copy the Bridge password (not your Proton account password). SmartInbox connects to Bridge at 127.0.0.1:1143. Stored locally in data/smartinbox.db only.

    Voice summary

    Delivery phrase audio in localrecordings/phrases/ is removed after this many hours (default 24). Use 0 to keep recordings indefinitely.

    When enabled, alerts may start with a short greeting like “Good morning Bill”, “Good evening Bill”, “Hello Bill”, or “Bill, you have mail” (time-of-day greetings use your configured timezone).

    Voice summary greets you by name (when enabled), announces who the email is from, then uses the LLM and your voice prompt for a brief spoken summary. Delivery mode still applies when voice summary is off.

    Voice prompt — used when Voice summary is on. Saved files live in prompts/.

    Demo mode

    Show sample emails for screenshots

    When On, the Inbox tab shows clearly fake [DEMO] messages so you can capture theme screenshots without exposing real mail. Polling, themes, summaries, and starring still work. Turn demo mode off to return to your live inbox.

    Calendar extraction

    How many emails Ollama processes in parallel during Check for Events on the Calendar tab. Higher values use more GPU/CPU and VRAM. With dual GPUs, try 6–8; reduce if you see timeouts or out-of-memory errors.

    Ollama GPU pins calendar extraction to a GPU index via Ollama's main_gpu option (inbox summaries are unchanged). Use 0 for GPU 0; clear the field for Ollama's automatic scheduling. The model is preloaded on that GPU before each scan.
